# 発表用トークスクリプト

## そのまま読む用

それでは発表させていただきます。

今回制作したのは、動画配信サービス風の自己紹介サイトです。

課題条件として、3ページ以上の構成、HTMLとCSSを使うこと、JavaScriptによるアニメーション表現を1ヶ所以上入れること、PCとスマホのレスポンシブ対応を行うことがありました。

今回のサイトでは、Home、History、Top Skills、Next Stageの4ページ構成にして、これらの条件を満たすように制作しています。

まず、全体のデザインについて説明します。

普通の履歴書のような自己紹介サイトだと、どうしても読み飛ばされやすいと考えました。

そこで、自分の人生や強みをエピソードとして読み進めてもらえるように、動画配信サービス風のUIにしています。

Homeでは自分の概要、Historyではこれまでの経歴、Top Skillsでは自分の強み、Next Stageでは今後の目標を見せる構成にしました。

次に、実装面について説明します。

まずHTMLでは、History、Top Skills、Next Stageの開閉カードに、HTML標準の `details` と `summary` を使っています。

`details` と `summary` を使った理由は、開閉できる情報をHTMLの構造として自然に表現できるからです。

JavaScriptを使わない状態でも最低限の開閉ができるので、構造としてもわかりやすいと考えました。

次にCSSについてです。

CSSでは、全体のデザイン、カードの見た目、サイドバー、レスポンシブ対応を管理しています。

PCでは左側にサイドバーを固定して、ページ間を移動しやすくしています。

一方で、スマホでは画面幅が狭くなるので、サイドバーを常時表示せず、ハンバーガーメニューで開閉する形にしました。

また、カードの配置も画面幅に応じて切り替えています。

PCではカードを横に並べて一覧性を出し、スマホでは読みやすいように縦1列にしています。

このように、PCの見た目をそのまま縮小するのではなく、画面サイズに合わせて操作しやすい形に変えています。

次にJavaScriptについて説明します。

JavaScriptでは、主にイントロアニメーション、スクロール時の表示アニメーション、スマホメニュー、カード開閉、次カードへの移動を実装しています。

まずHomeを開いた時に、最初に赤いTが表示されるイントロ演出を入れています。

これは `.intro-splash` というクラスが付いたイントロ用の要素を用意し、CSS側でアニメーションを動かしています。

JavaScript側では、`animationend` イベントでアニメーションが終わったタイミングを検知し、終了後に `.is-finished` クラスを付与しています。

これによって、イントロアニメーションが終わった後は、イントロ用の要素を非表示にして通常のHome画面を操作できるようにしています。

次に、スクロール時の表示アニメーションについてです。

ここでは、JavaScriptの `IntersectionObserver` APIを使っています。

監視対象のカードやセクションがビューポート、つまり画面内に入ったタイミングで `.is-visible` クラスを付与し、CSS側でふわっと表示されるようにしています。

スクロール位置を毎回手動で計算するのではなく、ブラウザの `IntersectionObserver` APIを使うことで、画面内に入ったタイミングを効率よく検知しています。

次に、各ページの工夫について説明します。

Historyページでは、各時代を開閉カードにしています。

本文を最初からすべて表示すると情報量が多く見えてしまうため、興味のある項目をクリックまたはタップして開く形にしました。

さらに、本文の最後に「中学生に続く」「高校生に続く」のようなボタンを置き、押すと次のエピソードが開くようにしています。

単なる年表ではなく、順番にエピソードを見ていく体験になるようにしました。

Top Skillsページでは、自分の強みを6つのカードに分けています。

PCでは2列で表示して一覧性を出しています。

スマホでは1列表示になるため、開いたカードの下に次のスキルへ進むボタンをJavaScriptで追加しています。

これによって、スマホでも下へ読み進めやすい導線にしています。

Next Stageページでは、短期、中期、長期の目標をカード形式で表示しています。

PCではカードを開くと大きく表示され、次の目標へ進めるようにしています。

スマホではカード内に前後移動ボタンを表示し、小さい画面でも操作しやすいようにしました。

全体を通して一番意識したのは、読み進めやすい導線です。

Homeの「はじめから見る」ボタン、Historyの次エピソードボタン、Top Skillsの次スキルボタン、Goalsの前後移動ボタンなど、ユーザーが迷わず次へ進めるように設計しました。

また、画像が重いという問題もあったため、最初に使っていたPNG画像をJPEGに変換し、画像フォルダ全体を軽量化しました。

画質を完全に優先するのではなく、自己紹介サイトとして十分きれいに見える範囲で、読み込みが重くなりすぎないようにしています。

最後に、AIの使い方についてです。

AIには、構成案や実装案、コード作成、文言整理の補助として使いました。

ただし、AIに丸投げしたのではなく、4ページ構成にすること、動画配信サービス風にすること、スマホで読みやすい導線にすること、自分の言葉のニュアンスを崩さないこと、画像を軽量化することは、自分で確認しながら判断しました。

また、HTML、CSS、JavaScriptがそれぞれ何を担当しているか、`details` と `summary`、`IntersectionObserver`、`animationend` などが何をしているかも、自分で説明できるように整理しました。

以上が今回制作した自己紹介サイトの説明です。

ありがとうございました。

## 詰まった時の言い換え

### `IntersectionObserver` の説明

専門寄り:

> スクロール時の表示アニメーションには、JavaScriptの `IntersectionObserver` APIを使っています。監視対象の要素がビューポート内に入ったタイミングで `.is-visible` クラスを付与し、CSS側でフェードインさせています。

簡単寄り:

> カードが画面内に入ったタイミングをJavaScriptで検知して、そのタイミングでふわっと表示されるようにしています。

### 赤いTのイントロ説明

専門寄り:

> `.intro-splash` にCSSアニメーションを設定し、JavaScriptで `animationend` イベントを検知しています。終了後に `.is-finished` クラスを付与して、イントロ要素を非表示にしています。

簡単寄り:

> Homeを開いた時に最初に出る赤いTの演出です。アニメーションが終わったら、JavaScriptでその画面を非表示にしています。

### `details` / `summary` の説明

専門寄り:

> 開閉カードにはHTML標準の `details` と `summary` を使っています。開閉UIをHTMLの構造として表現できるので、JavaScriptなしでも最低限の開閉ができます。

簡単寄り:

> カードを開いたり閉じたりする部分には、HTMLにもともと用意されている開閉用のタグを使っています。

### レスポンシブ対応の説明

専門寄り:

> CSSのメディアクエリで、画面幅に応じてレイアウトを切り替えています。PCでは2列や3列で一覧性を出し、スマホでは1列にして読みやすさを優先しています。

簡単寄り:

> PCでは横に並べて見やすくし、スマホでは画面が狭いので縦1列にしています。

## 発表直前チェック

- `IntersectionObserver` は「要素」ではなく「API」または「機能」と言う
- `animationend` は「アニメーション終了を検知するイベント」と言う
- `.is-visible` や `.is-finished` は「CSSクラス」と言う
- `CTA` と言う場合は「次へ進むボタン」と補足する
- 「Netflix」ではなく「動画配信サービス風」と言う
- 実装していない計測は「今後GTMを入れる場合にも見やすい構造を意識した」と言う
